> **چرا این ابزار ساخته شد؟** — ۸ اردیبهشت ۱۴۰۵
>
> در ایران، قطعی مکرر اینترنت و هزینه بالای VPN دسترسی به فایل‌های شخصی را سخت می‌کند.
> گیت‌هاب یکی از معدود سرویس‌هایی است که اغلب بدون VPN در دسترس است.
> این ربات به شما امکان می‌دهد فایل‌های سنگین درون تلگرام را (ویدیو، صدا، سند و …)
> مستقیماً در یک مخزن گیت‌هاب ذخیره کنید تا هر زمان، بدون VPN، از گیت‌هاب دانلود کنید.
>
> تنها نیاز: یک VPS ساعتی ارزان با حداقل ۱۰ گیگابایت SSD — و همین ربات.

---

## امکانات

- **پشتیبانی از همه انواع فایل** — سند، عکس، ویدیو، ویس، استیکر، انیمیشن، ویدیو مسیج
- **مسیریابی هوشمند:** فایل‌های زیر ۵۰ مگابایت از طریق Contents API و فایل‌های بزرگ‌تر از طریق **Git LFS** ذخیره می‌شوند
- **پاکسازی خودکار** — وقتی حجم مخزن از آستانه تعریف‌شده بگذرد، قدیمی‌ترین فایل‌ها حذف می‌شوند
- **کنترل دسترسی** — لیست سفید کاربران مجاز تلگرام
- **مدیریت تداخل فایل** — بازنویسی یا نسخه‌بندی خودکار (`file_1.ext`، `file_2.ext`، …)
- **مسیر آپلود سفارشی** — دستور `/setpath` برای هر کاربر
- **تلاش مجدد خودکار** — با تأخیر نمایی در صورت خطای شبکه یا محدودیت نرخ گیت‌هاب
- **اطلاع‌رسانی به ادمین** — ارسال خطاها به کاربر ادمین تنظیم‌شده
- **لاگ فایل چرخشی** — ۱۰ مگابایت × ۵ نسخه پشتیبان

## راهنمای نصب و راه‌اندازی

### ⚡ نصب یک‌خطی (سریع‌ترین روش)

کافی است این دستور را در ترمینال VPS اجرا کنید:

```bash
bash <(curl -sL https://raw.githubusercontent.com/GonnaTox/tg-github-uploader/main/setup.sh)
```

این دستور به‌صورت خودکار:

۱. دستور `apt update` را اجرا می‌کند و بسته‌های لازم را نصب می‌کند

۲. مخزن را کلون می‌کند

۳. محیط مجازی Python می‌سازد و وابستگی‌ها را نصب می‌کند

۴. فایل `.env` را با راهنمایی گام‌به‌گام می‌سازد

۵. بدون هیچ مرحله دستی **Git LFS را مستقیم روی مخزن گیتهاب شما فعال میکند**

۶. سرویس `systemd` را (در صورت تمایل) نصب می‌کند

> **پیش‌نیاز:** دسترسی `sudo` روی VPS و اتصال اینترنت.

قبل از اجرای اسکریپت، موارد زیر را آماده داشته باشید — ویزارد یک‌به‌یک از شما می‌خواهد:

- **[توکن ربات تلگرام](#-توکن-ربات-تلگرام)** — از @BotFather
- **[API ID و API Hash تلگرام](#-api-id-و-api-hash-تلگرام)** — از my.telegram.org
- **[توکن گیت‌هاب](#-توکن-گیت‌هاب)** — با دسترسی `repo`
- **[شناسه کاربری تلگرام](#-شناسه-کاربری-تلگرام)** — از @userinfobot

### ▶️ روش‌های اجرای ربات پس از نصب

**روش اول — systemd (پیشنهادی، پس از ریستارت VPS هم اجرا می‌شود):**

اگر در هنگام نصب، گزینه نصب سرویس systemd را انتخاب کرده‌اید:

```bash
sudo systemctl start tg-github-uploader   # شروع ربات
sudo systemctl status tg-github-uploader  # بررسی وضعیت
sudo journalctl -u tg-github-uploader -f  # مشاهده لاگ زنده
```

ربات پس از هر ریستارت یا کرش به‌صورت خودکار راه‌اندازی می‌شود.

**روش دوم — screen (ساده، بدون نیاز به systemd):**

```bash
screen -S tg2github
source .venv/bin/activate
python -m bot
```

برای جدا شدن بدون متوقف کردن ربات: `Ctrl+A` سپس `D`
برای بازگشت: `screen -r tg2github`

**روش سوم — اجرا مستقیم (فقط برای تست):**

```bash
source .venv/bin/activate
python -m bot
```

ربات با بستن ترمینال متوقف می‌شود.

### 🛠️ نصب دستی (روش کلاسیک)

#### ۱. دریافت کد و نصب وابستگی‌ها

```bash
sudo apt update
sudo apt install build-essential python3 python3-dev python3-venv python3-pip git git-lfs curl -y

git clone https://github.com/GonnaTox/tg-github-uploader.git
cd GitHub-uploader

python3 -m venv .venv
source .venv/bin/activate        # ویندوز: .venv\Scripts\activate
pip install -r requirements.txt
```

#### ۲. ساخت فایل تنظیمات

```bash
cp .env.example .env
nano .env   # یا با هر ویرایشگر دلخواه باز کنید
```

#### ۳. فعال‌سازی Git LFS در مخزن گیت‌هاب (برای فایل‌های بزرگ)

> **اگر از نصب یک‌خطی استفاده کردید، این مرحله به‌صورت خودکار انجام شده — نیازی به اجرای دستی نیست.**

اگر می‌خواهید فایل‌های بزرگ‌تر از ۵۰ مگابایت هم ذخیره شوند، باید یک‌بار Git LFS را در مخزن خود راه‌اندازی کنید — روی کامپیوتر شخصی خود (نه VPS) اجرا کنید:

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd YOUR_REPO

git lfs install
git lfs track "*"          # همه انواع فایل را تحت LFS قرار می‌دهد
git add .gitattributes
git commit -m "chore: enable Git LFS for all files"
git push
```

> اگر Git LFS روی کامپیوتر شما نصب نیست:
> - Ubuntu/Debian: `sudo apt install git-lfs`
> - macOS: `brew install git-lfs`
> - ویندوز: از [git-lfs.com](https://git-lfs.com) دانلود کنید

> 💡 اگر فقط فایل‌های زیر ۵۰ مگابایت آپلود می‌کنید، این مرحله اختیاری است.

#### ۴. اجرای ربات

به بخش [▶️ روش‌های اجرای ربات پس از نصب](#️-روش‌های-اجرای-ربات-پس-از-نصب) مراجعه کنید.

## راهنمای دریافت مقادیر لازم

### 🔑 API ID و API Hash تلگرام

این مقادیر برای Pyrogram (اتصال MTProto مستقیم) الزامی هستند.

۱. به [my.telegram.org](https://my.telegram.org) بروید و با شماره تلفن تلگرام خود وارد شوید.

۲. روی **"API development tools"** کلیک کنید.

۳. یک نام کوتاه برای اپلیکیشن بنویسید و روی **"Create application"** کلیک کنید.

۴. مقادیر `App api_id` (عدد) و `App api_hash` (رشته) را کپی کنید.

```
TELEGRAM_API_ID=12345678
TELEGRAM_API_HASH=abcdef1234567890abcdef1234567890
```

> ⚠️ این مقادیر را با کسی به اشتراک نگذارید و در فایل‌های عمومی کامیت نکنید.

---

### 🤖 توکن ربات تلگرام

۱. در تلگرام، [@BotFather](https://t.me/BotFather) را پیدا کنید و `/newbot` را ارسال کنید.

۲. یک نام و یک نام کاربری (که به `bot` ختم شود) انتخاب کنید.

۳. BotFather یک توکن مانند `1234567890:AAFxxx…` به شما می‌دهد — آن را کپی کنید.

```
TELEGRAM_BOT_TOKEN=1234567890:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

---

### 🔐 توکن گیت‌هاب

۱. در گیت‌هاب به **Settings → Developer settings → Personal access tokens → Tokens (classic)** بروید.

۲. روی **Generate new token (classic)** کلیک کنید.

۳. یک نام توضیحی بنویسید (مثال: `telegram-uploader-bot`) و یک تاریخ انقضا انتخاب کنید.

۴. در بخش **Select scopes** تیک **`repo`** را بزنید.

۵. روی **Generate token** کلیک کنید و توکن را همان لحظه کپی کنید — بعداً نشان داده نمی‌شود.

```
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

> ⚠️ این توکن را در هیچ فایلی کامیت نکنید و با کسی به اشتراک نگذارید.

---

### 👤 شناسه کاربری تلگرام

ربات [@userinfobot](https://t.me/userinfobot) را پیدا کنید و `/start` بزنید — شناسه عددی شما (مانند `123456789`) نمایش داده می‌شود.

```
ALLOWED_USER_IDS=123456789
ADMIN_USER_ID=123456789
```

> اگر `ALLOWED_USER_IDS` را خالی بگذارید، همه کاربران می‌توانند از ربات استفاده کنند.

## متغیرهای محیطی — جدول کامل

| متغیر | اجباری | پیش‌فرض | توضیح |
|---|---|---|---|
| `TELEGRAM_BOT_TOKEN` | ✅ | — | توکن ربات از BotFather |
| `TELEGRAM_API_ID` | ✅ | — | شناسه API از my.telegram.org |
| `TELEGRAM_API_HASH` | ✅ | — | هش API از my.telegram.org |
| `GITHUB_TOKEN` | ✅ | — | توکن دسترسی شخصی گیت‌هاب |
| `GITHUB_OWNER` | ✅ | — | نام کاربری یا سازمان گیت‌هاب |
| `GITHUB_REPO` | ✅ | — | نام مخزن |
| `GITHUB_BRANCH` | ❌ | `main` | شاخه هدف |
| `UPLOAD_BASE_PATH` | ❌ | `uploads` | پوشه ریشه در مخزن |
| `FILE_CONFLICT_STRATEGY` | ❌ | `version` | `overwrite` یا `version` |
| `LFS_THRESHOLD_MB` | ❌ | `50` | فایل‌های بزرگ‌تر از این مقدار (مگابایت) از طریق Git LFS آپلود می‌شوند |
| `CLEANUP_ENABLED` | ❌ | `false` | پاکسازی خودکار فایل‌های قدیمی را فعال می‌کند |
| `CLEANUP_MAX_REPO_MB` | ❌ | `800` | آستانه حجم مخزن (مگابایت) برای شروع پاکسازی |
| `CLEANUP_KEEP_LATEST` | ❌ | `10` | تعداد جدیدترین فایل‌هایی که هیچ‌گاه حذف نمی‌شوند |
| `ALLOWED_USER_IDS` | ❌ | *(همه)* | شناسه‌های مجاز با کاما جدا |
| `ADMIN_USER_ID` | ❌ | *(هیچ)* | شناسه ادمین برای دریافت خطاها |
| `LOG_LEVEL` | ❌ | `INFO` | سطح لاگ: `DEBUG`، `INFO`، `WARNING` |
| `LOG_FILE` | ❌ | `logs/bot.log` | مسیر فایل لاگ |

## محدودیت حجم فایل و مسیریابی هوشمند

| لایه | حداکثر |
|---|---|
| Pyrogram / MTProto (دانلود از تلگرام) | **۲۰۰۰ مگابایت** |
| GitHub Contents API (فایل‌های زیر آستانه) | **۱۰۰ مگابایت** |
| Git LFS (فایل‌های بزرگ‌تر از آستانه) | **۲ گیگابایت** (در حساب رایگان) |

ربات به صورت خودکار تصمیم می‌گیرد:
- فایل‌های کوچک‌تر از `LFS_THRESHOLD_MB` (پیش‌فرض ۵۰ مگابایت) → **Contents API**
- فایل‌های بزرگ‌تر → **Git LFS**

## ⚠️ محدودیت‌ها و ریسک‌های Git LFS — قبل از استفاده بخوانید

Git LFS تنها راه عملی برای ذخیره فایل‌های بزرگ روی گیت‌هاب است، اما در حساب رایگان محدودیت‌های سختی دارد که باید از آن‌ها آگاه باشید.

### سقف رایگان (هر حساب، هر ماه)

| منبع | سهمیه رایگان |
|---|---|
| **فضای ذخیره‌سازی** | **۱۰ گیگابایت** کل در همه مخازن |
| **پهنای باند** (دانلود) | **۱۰ گیگابایت** در ماه |

فضای ذخیره به ازای هر فایلی که push می‌کنید محاسبه می‌شود — حذف فایل در اواسط ماه محاسبه را کاهش نمی‌دهد. پهنای باند هر بار که **هر کسی** فایل LFS را دانلود کند مصرف می‌شود.

**وقتی به سقف رسیدید:**
- LFS تا پایان همان ماه برای حساب شما غیرفعال می‌شود.
- آپلود و دانلود فایل‌های LFS با خطا مواجه می‌شوند.
- در اول ماه بعد سهمیه ریست می‌شود.

**خرید فضای اضافه: ماهانه ۵ دلار به ازای ۵۰ گیگابایت فضا + ۵۰ گیگابایت پهنای باند.**

### ⚠️ هشدارهای مهم

> **برای این ربات یک حساب گیت‌هاب جداگانه بسازید.**
>
> از حساب اصلی توسعه‌دهی خود استفاده نکنید. گیت‌هاب می‌تواند حساب‌ها را به دلیل فعالیت‌های خودکار غیرمعمول بدون اطلاع قبلی مسدود کند، و پاسخ پشتیبانی آن‌ها گاهی هفته‌ها طول می‌کشد. اگر حساب ربات تعلیق شود، حساب اصلی شما — و تمام مخازن کدتان — در امان می‌مانند.

- **مخزن آپلود را خصوصی نگه دارید.** یک مخزن عمومی با LFS یعنی هر کسی می‌تواند فایل‌هایتان را دانلود کند و سهمیه ۱۰ گیگابایت پهنای باند ماهانه شما را در چند دقیقه تمام کند.
- **مصرف را رصد کنید** در: GitHub → Settings → Billing & plans → Git LFS Data.
- سهمیه رایگان در اول هر ماه ریست می‌شود، اما **سهمیه استفاده‌نشده به ماه بعد منتقل نمی‌شود**.

## پاکسازی خودکار (مناسب VPS ارزان)

برای VPS‌هایی با SSD کوچک (مثلاً ۱۰ گیگابایت)، می‌توانید پاکسازی خودکار را فعال کنید.
وقتی حجم کل فایل‌های آپلودشده از `CLEANUP_MAX_REPO_MB` بگذرد، ربات قدیمی‌ترین فایل‌ها را
حذف می‌کند و `CLEANUP_KEEP_LATEST` جدیدترین فایل را همیشه نگه می‌دارد.

```env
CLEANUP_ENABLED=true
CLEANUP_MAX_REPO_MB=2048
CLEANUP_KEEP_LATEST=10
```
